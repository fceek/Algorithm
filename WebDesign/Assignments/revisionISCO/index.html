7a Data Representation

- Intro -

How was data stored in memory/computers?

Numbers were stored in binary or hex; characters and text were stored by using character sets; images were stored as bitmaps.

- Numbers -

How positive integers are stored in computers in binary?

Binary is to use 0 and 1 to represent everything which is easy to be interpreted by computers. After decimal numbers been converted to binary, they could be saved in memory every 8 bits.

Converting binary to decimal

Simply add every non-0 digit's value(or say, weight) together to get the decimal.

For this example, 10110101 is 128+32+16+4+1 = 181

Converting decimal to binary

Try to fit every digit's value to the decimal number from higher to lower, note the digit that fits as 1 and subtract the value from decimal number.

Follow this loop till the number becomes 0.

Take the same example, the process is: 181 - 128 = 53, so the 8th digit whose value is 128 is 1; 53 - 32 = 21, 6th digit is 1...

Sign and Magnitude / Two's Complement

Sign and Magnitude

Use the highest digit to represent sign. 0 is positive, 1 is negative. Remaining part is just the normal binary number.

Two's Complement

The sign part is the same, 0 for + and 1 for -. However, when the sign is -, the number part should be all reversed and added 1.

For example, +117 is stored as 01110101 in both methods. But in sign and magnitude, -117 is 11110101; in two's complement, -117 is 10001011, which is the reversed version 10001010 + 1.

Represent Numbers in Hex

One digit in Hex equals to 4 digits in binary. Example: 73B = 0111 0011 1011. So just make them binary and use converting between binary and decimal.

- Text -

Character Sets: ASCII and Unicode

Character sets define relationships between characters and binary numbers so that characters can be saved in memory. ASCII uses 8 dights to represent a character while Unicode uses 16, that makes Unicode files larger but can support more characters.

- Image -

Bitmap

First we declare the index of each color, like 0 for white and 1 for black. Then we store the image by store the color number of each pixel.

Color depth

If there are blue and red color other than black and white, we need another digit to mark them. 1 digit can represent 2 colors, 2 can represent 4, 3 for 8... The number of digits is color depth.

An image also contains metadata, that records its name, arthor, type, date or other information.

Calculate size of a bitmap

[width(in pixels) x height(in pixels) x color depth + size of metadata] / 8.
